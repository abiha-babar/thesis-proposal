\section{Proposed Method}
\label{sec:method}

\subsection{What This Study Does}

This study sets up OpenClaw \cite{openclaw_2025} and runs the same set of tasks twice: once with all skills loaded into the agent at the start, and once with skills loaded one at a time as they are needed. Everything else stays the same both times --- the same tasks, the same AI models, and the same way of checking the results. The only thing that changes is how the skills are given to the agent.

By keeping everything else identical, any difference in results can only be explained by the one thing that changed: the way skills were loaded.

\subsection{The Tools and Software Used}

The study uses four things:

\begin{itemize}
    \item \textbf{OpenClaw} \cite{openclaw_2025}: The agent framework used to run all tasks. OpenClaw handles the agent loop, loads skill files, and writes a detailed log of everything the agent does. Those logs are what the measurements come from.
    \item \textbf{Ollama} \cite{ollama_2024}: A free tool that runs AI models locally on a laptop. It handles setup automatically. Two local models will be used: \texttt{gemma4:4b} by Google DeepMind \cite{gemma4_2026} and \texttt{qwen3.5:4b} by Alibaba \cite{qwen3_2025}.
    \item \textbf{Gemini 3.1 Pro} \cite{google_gemini_2025}: A cloud-based AI model by Google, used as the main cloud reference. It shows whether the findings hold for large, powerful models too.
    \item \textbf{GPT-4o} \cite{openai_gpt4o_2024}: A second cloud model by OpenAI, used to check whether the findings are the same across different cloud providers.
    \item \textbf{Python}: Used to read the log files that OpenClaw writes and calculate all the numbers. No extra frameworks are needed.
\end{itemize}

\subsection{The Two Setups}

\subsubsection*{Setup A --- All Skills Loaded at Once (the standard way)}

In Setup A, all 20 skills are written out as a flat list and added to the agent's instructions at the very beginning of every task. The agent reads through the whole list before each step and picks the skill it thinks it needs. This is how most AI agents work today.

\subsubsection*{Setup B --- Skills Loaded One at a Time (using OpenClaw)}

In Setup B, the agent starts with only a short list of skill names and one-line descriptions --- nothing more. When the agent decides it needs a specific skill, OpenClaw loads the full instructions for that skill. Once that skill is finished, those instructions are removed before the next step begins. The agent never has more than one skill's full instructions in its memory at a time.

\subsection{The Skill Files}

Twenty skill files will be written for this study, each stored as a \texttt{SKILL.md} file. They cover three areas of everyday tasks that OpenClaw is designed for:

\begin{enumerate}
    \item \textbf{Communication (6 skills):} Drafting an email reply; composing a meeting invitation; summarising unread messages; writing a short chat message; generating a weekly update; and formatting a written response from notes.

    \item \textbf{Research and Information (8 skills):} Searching the web for a topic; summarising an article; looking at two options side by side; checking whether a claim is true online; monitoring a keyword for new results; generating a research briefing from multiple sources; extracting key numbers from a page; and ranking a list of options.

    \item \textbf{Task and Planning (6 skills):} Creating a to-do list from a description of goals; setting a reminder; generating a morning briefing; reporting what is done and what remains on a goal; writing a project status update; and producing an action-item list from meeting notes.
\end{enumerate}

Each skill file has three parts: (1) when the skill should be used; (2) the steps the agent must follow; and (3) how the agent knows when the skill is finished. In Setup A, the same 20 skills are written as a flat JSON list instead.

The study will be run twice: once with 10 skills and once with all 20, to see whether the gap between the two setups grows as more skills are added.

\subsection{The Tasks}

Forty tasks will be used, split into three levels:

\begin{itemize}
    \item \textbf{Level 1 --- One skill needed (15 tasks):} The agent only needs one skill to finish the task. Examples: \textit{``Search the web for the latest news on autonomous AI and give me three key points.''} Or: \textit{``Draft a polite email declining a meeting invitation.''} Or: \textit{``Make a to-do list from these goals.''}

    \item \textbf{Level 2 --- Two skills needed (15 tasks):} The agent needs two skills, one after the other. Examples: \textit{``Search for the top five project management tools, then draft an email recommending the best one to my team.''} Or: \textit{``Check whether this claim is true online, then write a reply to the sender with the answer.''}

    \item \textbf{Level 3 --- Three or more skills needed (10 tasks):} The agent needs several skills and must make a decision in the middle based on what it found. Examples: \textit{``Find the three most talked-about AI papers this week, summarise each one in two sentences, and write a morning briefing for my team.''} Or: \textit{``Check whether this event is still happening online, update my to-do list, and send a message to the group with any changes.''}
\end{itemize}

Every task is run in both setups and with all four AI models, repeated three times each. That gives a total of $40 \times 2 \times 4 \times 3 = 960$ runs.

\subsection{What Gets Measured}

\begin{itemize}
    \item \textbf{Did the agent finish the task correctly?} The final answer is checked against a known correct answer. Text tasks use the ANLS* scoring method \cite{anlsstar_2024}, which gives partial credit for nearly-correct answers. Other tasks get a simple pass or fail.

    \item \textbf{Did the agent try to use a skill that does not exist?} Every time the agent calls a skill name that is not in the library, or uses a skill in a situation it is not meant for, that is counted as an error. These are read directly from the OpenClaw log files.

    \item \textbf{Did the agent follow the steps inside the skill file?} The steps the agent actually took are checked against the steps written in the skill file. A score is given based on how many steps were followed in the right order.

    \item \textbf{How many tokens did the agent use?} Token counts come from the Ollama server logs for local models and from the API response for cloud models. This shows directly how much memory the agent was using at each step.

    \item \textbf{How long did each task take?} Time is measured from when the task was sent to when the final answer came back.

    \item \textbf{How much did each run cost?} For cloud models, cost comes from the API billing (tokens used multiplied by the price per million tokens). For local models, power is measured using \texttt{powermetrics} on Mac or \texttt{powertop} on Linux \cite{tokenpowerbench_2024} and converted to a cost using the local electricity rate.
\end{itemize}

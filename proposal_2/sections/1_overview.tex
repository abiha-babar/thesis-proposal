\section{Overview}
\label{sec:overview}

The deployment of autonomous artificial intelligence agents has accelerated substantially in the period from 2024 to 2026, driven by the maturation of open-source orchestration frameworks and the widespread availability of locally deployable language models \cite{wang2024survey}. These agents are no longer confined to generating text in response to a single user query; they are now expected to plan multi-step workflows, invoke external tools, read and write files, and recover from intermediate failures --- all without continuous human supervision \cite{yao2022react}. As the operational scope of autonomous agents expands, a fundamental architectural question has emerged: how should an agent's available capabilities be structured and presented to the underlying language model?

The predominant approach in current deployments, referred to in this study as \textit{flat tool use}, exposes the complete set of available functions to the agent as a single, unified list within the system prompt \cite{schick2023toolformer}. While straightforward to implement, this paradigm has been shown to degrade agent reasoning quality as the capability set grows, because irrelevant tool definitions occupy a disproportionate share of the available context window, leaving less capacity for task-specific reasoning \cite{patil2023gorilla, shi2024context}. The problem is compounded for locally deployed small language models, whose context windows are considerably shorter than those of frontier cloud-based systems \cite{gemma4_2026}.

An alternative paradigm, implemented in the open-source autonomous agent framework OpenClaw \cite{openclaw_2025} and formalised in recent research on executable agent policies \cite{wang2024executable}, addresses this limitation through an architecture known as \textit{Progressive Disclosure}. Rather than loading all capability definitions simultaneously, this approach structures agent skills into a three-tier hierarchy: the agent first receives only a compact index of skill names and one-line descriptions; upon identifying a relevant skill, it retrieves that skill's full procedural instructions from a local \texttt{SKILL.md} file; and it accesses the underlying executable scripts or external service integrations only at the moment of action. Upon task completion, the loaded skill content is removed from the active context before the next reasoning cycle begins. This design is intended to reduce context saturation, constrain agent behaviour through explicit procedural policies, and lower the incidence of hallucinated tool invocations.

Despite the growing adoption of this architecture in production autonomous systems, no published empirical study has directly compared flat tool use against progressive skill disclosure under controlled experimental conditions. The performance claims made by framework developers remain unvalidated against independent benchmarks, and the conditions under which progressive disclosure provides the greatest benefit --- such as skill library size, task complexity, and underlying model capability --- are entirely unknown. This thesis addresses that gap through a controlled, reproducible experiment using OpenClaw \cite{openclaw_2025} as the agent framework, Ollama \cite{ollama_2024} for local model inference, and the Gemini 3.1 Pro \cite{google_gemini_2025} and GPT-4o \cite{openai_gpt4o_2024} APIs as cloud-based evaluation baselines.

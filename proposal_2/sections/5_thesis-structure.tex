\section{Overall Structure of the Proposed Thesis}
\label{sec:structure}

The final written thesis will be divided into the following chapters:

\begin{enumerate}
    \item \textbf{Introduction.} Explains the problem with flat tool lists in AI agents, why progressive disclosure is a proposed solution, and what this study is trying to find out. Gives the reader a clear picture of what the whole thesis is about.

    \item \textbf{Literature Review.} Summarises what other researchers have already found about tool use in AI agents, why long tool lists cause problems, and how trajectory quality should be measured. Explains exactly what is still missing from existing research and presents the three questions this thesis will answer.

    \item \textbf{Methodology.} Describes exactly how the experiment was set up. Explains the two agent conditions (flat tool list versus progressive disclosure via OpenClaw), how the 20 skill files were built, how the 40 tasks were designed across three difficulty levels, which models were tested (Gemma~4, Qwen~3.5 via Ollama; Gemini~3.1~Pro and GPT-4o via cloud APIs), and how each metric was measured from the OpenClaw JSONL logs.

    \item \textbf{Results.} Shows all the data collected from the 960 agent runs: how accurate each setup was, how many hallucinated tool calls occurred, how well the agents followed skill instructions, how many tokens were used per task, how fast each run was, and what each run cost. Charts and tables will be included to make the results easy to read.

    \item \textbf{Discussion.} Explains what the results mean. Discusses whether the findings answer the three research questions, what the limits of the study are, and what future researchers could look into next.

    \item \textbf{Conclusion.} A short summary of the most important findings and what they mean for anyone building or using autonomous AI agents.

    \item \textbf{References.} A full list of all research papers and sources used in the thesis.

    \item \textbf{Appendices.} Supporting material including: the \texttt{SKILL.md} template used for all 20 skills, the full task list with answer keys, the Python scripts used for reading logs and computing metrics, the full raw results tables, and the power measurement setup for local models.
\end{enumerate}

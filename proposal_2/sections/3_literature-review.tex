\section{Literature Review}
\label{sec:literature}

\subsection{Existing Research}

Prior research relevant to this thesis spans four distinct but interconnected areas: the evolution of tool use in language model agents, the architecture of agentic skill systems, evaluation methodologies for autonomous agents, and the practical deployment of autonomous frameworks.

\subsubsection*{Tool Use in Language Model Agents}

The foundational work on equipping language models with the ability to call external tools was carried out by Schick et al.\ \cite{schick2023toolformer}, who introduced Toolformer---a method for training models to decide when and how to invoke APIs by generating API calls inline within their output. Patil et al.\ \cite{patil2023gorilla} subsequently demonstrated that tool accuracy degrades significantly when the number of available APIs grows large, introducing the Gorilla benchmark to measure this effect. Taken together, these studies establish the core problem that the skills paradigm attempts to solve: flat tool lists harm agent performance as the capability set expands.

Qin et al.\ \cite{qin2023toolllm} extended this line of research with ToolBench, a large-scale benchmark of over 16,000 real-world APIs, showing that state-of-the-art models fail on multi-step tool composition tasks even when individual tool selection is reliable. This finding underscores the importance of procedural structure in guiding agent behavior---the exact function served by the execution policy component of an agentic skill.

\subsubsection*{Agentic Skill Architectures}

Wang et al.\ \cite{wang2024executable} introduced the concept of executable policies for embodied agents, in which behavioral constraints and procedural instructions are stored externally and loaded on demand. This work is the closest academic precursor to the SKILL.md architecture used in open-source autonomous frameworks: it demonstrates that externalized procedural knowledge improves task completion rates over pure in-context reasoning.

The OpenClaw framework \cite{openclaw_2025} represents the primary production implementation of progressive disclosure in the agent-skills paradigm. Its architecture organizes capabilities into a three-tier hierarchy---metadata, instructional, and execution tiers---and exposes them to the underlying language model incrementally based on task relevance. The heartbeat scheduling mechanism and persistent SQLite memory used by OpenClaw distinguish it from stateless frameworks such as LangGraph \cite{langgraph_2024} and CrewAI \cite{crewai_2024}, which do not maintain persistent state between invocation cycles.

\subsubsection*{Agent Evaluation Methodologies}

Liu et al.\ \cite{liu2023agentbench} introduced AgentBench, the first standardized benchmark for evaluating language model agents across diverse interactive environments. Their work establishes the methodological distinction between outcome metrics---which assess only whether the final state is correct---and trajectory metrics, which evaluate the entire reasoning and tool-invocation sequence. This distinction is central to the current study, as the primary claim of progressive disclosure is not merely that it produces better outcomes, but that it produces more efficient and compliant execution paths.

Yao et al.\ \cite{yao2022react} introduced the ReAct framework, which interleaves reasoning steps with action calls to produce traceable, interpretable agent trajectories. ReAct's trace format provides the methodological foundation for trajectory-level evaluation used in this thesis.

Zhou et al.\ \cite{zhou2023webarena} created WebArena, a realistic benchmark of long-horizon web navigation tasks, finding that even frontier models succeed on fewer than 15\% of tasks when evaluated on full trajectories rather than final outcomes. This result highlights the severity of the evaluation gap between outcome-only and trajectory-aware assessment---and motivates the dual-metric evaluation design used in this study.

\subsubsection*{Context Window Saturation and Retrieval}

Shi et al.\ \cite{shi2024context} empirically demonstrated that irrelevant information in the context window consistently degrades large language model reasoning performance. This finding provides a theoretical mechanism for why progressive disclosure should improve agent accuracy: by loading only the relevant skill into context rather than all available tools, the agent's reasoning operates on a cleaner, less noisy prompt. Loss et al.\ \cite{liu2024lost} further showed that language models disproportionately attend to information at the beginning and end of long contexts, suppressing the processing of information in the middle---a phenomenon they term the ``lost in the middle'' effect---which implies that large flat tool lists are particularly harmful when the relevant tool definition appears near the center of the prompt.

\subsection{Research Gap}

The existing literature establishes two important facts. First, flat tool lists harm agent performance as the tool set grows, because irrelevant context degrades LLM reasoning and because flat lists provide no procedural guidance during execution \cite{patil2023gorilla, shi2024context}. Second, trajectory-level evaluation is necessary to make meaningful claims about agent reliability, since outcome-only metrics mask inefficient, non-compliant, or unsafe execution paths \cite{liu2023agentbench, zhou2023webarena}. However, no existing study has directly compared a flat tool-use architecture against a progressive skill disclosure architecture on matched tasks, using trajectory-level metrics to assess whether the structural change produces meaningful improvements in efficiency, compliance, and hallucination rate. This thesis fills that gap.

\subsection{Research Questions}

Based on the literature above, this thesis will answer three specific questions:

\begin{itemize}
    \item \textbf{Question 1:} Does an agent operating under a progressive skill disclosure architecture produce significantly fewer hallucinated tool calls and higher trajectory compliance scores than the same agent operating under a flat tool-use architecture, across tasks of varying complexity?
    \item \textbf{Question 2:} How much does progressive disclosure reduce prompt token consumption and total inference cost compared to flat tool exposure, and how does this reduction scale with the size of the skill library?
    \item \textbf{Question 3:} Under what conditions---defined by skill library size, task length, and model capability---does progressive disclosure provide the greatest improvement in outcome accuracy and trajectory quality?
\end{itemize}
